Git and Github. How it work?


clear
очистить консоль (powershell & git bash)
ls
показывает список папок и файлов в текущей дериктории
pwd
показыввает наше текущее расположение
cd
открыть паку в текущей директории (перейти на 1 уровень вниз)
cd ..
перейти на 1 уровень вверх относительно текущей директоии
mkdir new-folder
создать папку с именем “new-folder”
touch index.html
создать файл index.html
nano index.html
редактировать файл index.html
git --version
проверяет установлен ли Git и его версию
git init
Инициализация нового репозитория для проекта (выполняется 1 раз для проекта, в корне поректа)
git status
Показывает текущий состояние файлов в репозитории (отслеживаемые / неотслеживаемые)
git add index.html
происходит отслеживание файла index.html (его изменений) в репозитории
git add .
происходит отслеживание всех файлов (их изменений) в репозитории
git rm --cached info.txt
прекратить отслеживание изменений в файле info.txt
git commit -m “создал файл index.html”
создает новый коммит (новую запись в истории репозитория) с именем “создал файл index.html”. Сохраняется состояние файлов репозитория на момент выполнения команды git add (git add .). Следовательно, пред выполением команды git commit -m необходимо обязательно выполнить команду git add (git add .).
.gitignore
файл (создающийся в корне текущего репозитория) в котором перечисляются имена файлов и папок которые не нужно отслеживать (сохранять). Указав имя папки, все содержимое папки тоже не отслеживается.
git branch
показывает список веток
git branch -M main
Переименовывает текущую ветку на имя “main”
git branch readme
создает новую ветку с именем “readme”
git branch -D readme
удаляет ветку с именем “readme”
git checkout readme
переключается на другую ветку с именем “readme”
git checkout -b project2
создает новую ветку “project2” и сразу переключается на нее
git merge project2
совмещает текущую ветку с файлами ветки “project2”
git config --global user.name
Показывает имя пользователя
git config --global user.name 'new user'
Изменяет имя пользователя
git config --global user.email
Показывает email пользователя
git config --global user.email 'test@mail.ru'
Изменяет email пользователя
git remote add origin git@github.com...
(Формат авторизации – SSH) Добавляем связь между удаленным и локальным репозиторием
git remote remove origin
Удаляем связь между удаленным и локальным репозиторием
git push orogin master
заливает текущие локальные коммиты в удаленный репозиторий
git clone git@github.com...
клонирует проект с удаленного репозитория “git@github.com...”
git pull
забирает изменения с удаленного репозитория в локальный
